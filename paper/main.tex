\begin{abstract}
By proposing network-theoretic methods as a mode of inquiry for formalized mathematics, we attempt to distinguish actionable software engineering targets from underlying mathematical structures. We see large-scale formal libraries like Lean~4's \texttt{Mathlib} as \emph{transitional artifacts}: they simultaneously encode inherited mathematical structure and the organizational processes of the community that builds them. To study this duality, we extract multi-layer dependency graphs, spanning declaration, module, and namespace levels. We then introduce graph decompositions that helps to disentangle these dimensions, isolating compiler-derived edges (74.2\%) from human-authored ones, proof-driven dependencies (43.9\%) from statement-driven ones, and language infrastructure from mathematical hubs. These networks are objective structures driving engineering interventions (e.g., the \texttt{shake} linter, typeclass caching, parallel compilation). Our empirical analysis uses longitudinal data across four \texttt{Mathlib} snapshots (2024--2026). By doing so, we identify latent structural bottlenecks currently lacking mitigations, specifically, 1.6\% import utilization, definitional heights up to 60, 44.8\% terminal theorems. Finally, we identify intrinsic mathematical signatures like dense cross-namespace coupling (50.9\%) and hierarchical layering (153, 84, and 8 layers). 
\end{abstract}

\section{Network Analysis Details}\label{app:supplementary}

Section~\ref{sec:cross_level} then overlays the three levels, quantifying how file boundaries, namespace boundaries, and logical dependencies align and diverge.
