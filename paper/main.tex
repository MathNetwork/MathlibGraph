\begin{abstract}
The formalization of mathematics transforms latent logical dependencies into explicit, machine-analyzable networks. However, the organization of these libraries is fundamentally driven by human cognitive habits, collaborative workflows, and tooling constraints. To quantify the divergence between the human organizational process and the inherited mathematical structure, we conduct a multi-layer network analysis of Lean~4's \texttt{Mathlib}. By mapping dependencies between files, namespaces, and theorems, we observe that file boundaries capture only 10.7\% of logical connections, and namespace boundaries capture only about 14\% at finer levels. We identify a distinct contrast where the human-organized file graph is deep (153 layers) while the underlying logical graph is much shallower (84 layers). Furthermore, we find that 17.5\% of imports are redundant links included for development convenience rather than logical necessity. These observations help quantifying the extent to which formal libraries reflect human organizational choices, yielding structural metrics for AI training and library maintenance.
\end{abstract}

%\xinze{The formalization of mathematics is translating centuries of implicit human reasoning into explicit, machine-verifiable code. As collaborative libraries like \texttt{Mathlib} reach unprecedented scales, they offer a unique empirical record of the macroscopic architecture of mathematical knowledge. However, it remains unclear how the strict logical dependencies required for formal verification interact with the cognitive and organizational strategies of the human communities that build them. Here, we analyze the network topology of \texttt{Mathlib} to quantify the relationship between human file organization and intrinsic mathematical logic. By comparing the macroscopic \emph{module import graph} ($N \approx 7.6 \times 10^3$) with the microscopic \emph{declaration dependency graph} ($N \approx 3.1 \times 10^5$), we find that 17.5\% of file imports are logically redundant---even after Mathlib's \texttt{shake} linter has already pruned certain redundant imports in CI. Rather than reflecting compiler necessity, these redundancies serve as development ergonomics: they ease refactoring and future extensibility of library files~\cite{baanen2025}. Furthermore, while directory structures effectively compartmentalize broad fields like Algebra and Analysis, fine-grained theorem dependencies frequently transcend these boundaries. The library exhibits a heavy-tailed citation distribution, bounded proof complexity, a remarkably deep hierarchy (153 layers), and a hub-dominated architecture vulnerable to targeted disruption. Our snapshot (February 2026) captures a transitional state: Lean~4's module system~\cite{lean4modules} (introduced November 2025) distinguishes \texttt{public import} from \texttt{import}, and as private imports are adopted the import graph structure will change significantly. By establishing a quantitative baseline for formalized library organization, these metrics inform maintenance prioritization and AI-assisted premise selection. Ultimately, our findings suggest that future proof assistants and search tools might benefit from architecturally decoupling the minimal dependency graphs required for verification from the semantically rich structures optimized for human interaction.}
