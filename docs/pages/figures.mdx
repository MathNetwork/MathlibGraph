# Figure Generation

All paper figures are generated by Python scripts in `src/plots/`. They share a unified style via `plots/plot_style.py`.

## Shared Style

`src/plots/plot_style.py` defines colors, figure sizes, and matplotlib rcParams used by every plotting script:

```python
from plot_style import setup_style, COLORS, FIGSIZE_SINGLE, FIGSIZE_DOUBLE

COLORS = setup_style()  # returns the color dict and applies rcParams
```

### Color Palette

| Key | Hex | Used for |
|-----|-----|----------|
| `primary` | `#2E86AB` | Module-level data (blue) |
| `secondary` | `#E94F37` | Declaration-level data (red) |
| `tertiary` | `#2E8B57` | Namespace-level data (green) |
| `quaternary` | `#F5A623` | Power-law fit lines (gold) |
| `grey` | `#888888` | Baselines, annotations |

### Figure Sizes

| Constant | Size (inches) | Use case |
|----------|--------------|----------|
| `FIGSIZE_SINGLE` | 6 x 4 | Single plot |
| `FIGSIZE_DOUBLE` | 12 x 5 | Two subplots side by side |
| `FIGSIZE_TRIPLE` | 15 x 4.5 | Three subplots |
| `FIGSIZE_HEATMAP` | 12 x 5.5 | Heatmap with labels |

### Style Settings

- Font: serif (STIX Two Text / STIXGeneral) to match LaTeX Computer Modern
- Output: PDF at 300 DPI
- Spines: top and right removed
- Grid: off by default (scripts enable per-axis as needed)

## Script Reference

### `plot_containment.py`

Generates the containment decay curve (Figure in Section 6). Uses hardcoded data from prior analysis runs -- no HuggingFace download needed.

```bash
python src/plots/plot_containment.py
```

**Output:** `paper/analysis/containment_curve.pdf`

Plots namespace containment (%) at depths k=1..6 (green line), file-system levels (blue squares for top-level directory and file module), and a 9.6% same-file baseline (grey dashed).

---

### `plot_dag_depth.py`

Computes topological layer widths (DAG depth profiles) for all three graph levels.

```bash
python src/plots/plot_dag_depth.py
```

**Output:**
- `paper/analysis/thm_dag_structure.pdf` -- Declaration-level DAG width
- `paper/analysis/ns_dag_structure.pdf` -- Namespace-level condensed DAG (with giant SCC annotation)
- `paper/analysis/dag_structure.pdf` -- Module-level DAG (raw + transitive reduction)

**Data sources:** HuggingFace (declarations/edges) + local `mathlib4/Mathlib/` (module imports).

The namespace graph is not a DAG, so the script condenses SCCs first, then computes layers on the condensation.

---

### `plot_namespace_degree_dist.py`

Plots in-degree and out-degree distributions of the namespace graph G_ns^(2) on log-log axes, with power-law reference lines.

```bash
python src/plots/plot_namespace_degree_dist.py
```

**Output:** `paper/analysis/ns_degree_distribution.pdf`

**Data source:** HuggingFace. Builds G_ns^(2) by truncating declaration names to depth 2, then counting unique namespace-pair edges.

---

### `regen_figures.py`

Regenerates the declaration-level degree distribution and robustness curve from pre-computed CSVs.

```bash
python src/plots/regen_figures.py
```

**Output:**
- `paper/analysis/thm_degree_distribution.pdf`
- `paper/analysis/thm_robustness_curve.pdf`

**Prerequisites:** Requires `output/powerlaw_fit_results.csv` and `output/robustness_data.csv` from a prior `run_degree_fit.py` / `run_cascade_robustness.py` run.

---

### `plot_robustness_curves.py`

Simulates progressive node removal (random vs. PageRank-targeted) for module and namespace levels.

```bash
python src/plots/plot_robustness_curves.py
```

**Output:**
- `paper/analysis/module_robustness_curve.pdf`
- `paper/analysis/ns_robustness_curve.pdf`

**Data sources:** Local `mathlib4/Mathlib/` for module graph, HuggingFace for namespace graph. Computes PageRank, then measures largest WCC fraction as nodes are removed at fractions 1%--50%.

---

### `plot_centrality_scatter.py`

Generates pairwise centrality scatter plots (in-degree vs PageRank, in-degree vs betweenness, betweenness vs PageRank) for declaration and namespace levels.

```bash
python src/plots/plot_centrality_scatter.py
```

**Output** (6 files):
- `paper/analysis/thm_centrality_indeg_pr.pdf`
- `paper/analysis/thm_centrality_indeg_betw.pdf`
- `paper/analysis/thm_centrality_betw_pr.pdf`
- `paper/analysis/ns_centrality_indeg_pr.pdf`
- `paper/analysis/ns_centrality_indeg_betw.pdf`
- `paper/analysis/ns_centrality_betw_pr.pdf`

**Data source:** HuggingFace. Declaration betweenness uses k=500 sampled approximation; namespace betweenness uses k=300.

---

### `plot_community_figures.py`

Generates Louvain community visualizations for all three graph levels: aggregation graphs (node = community, edge = inter-community flow) and community-vs-category heatmaps.

```bash
python src/plots/plot_community_figures.py
```

**Output** (6 files in `paper/figures/`):
- `community_module_graph.pdf` / `community_module_heatmap.pdf`
- `community_decl_graph.pdf` / `community_decl_heatmap.pdf`
- `community_ns_graph.pdf` / `community_ns_heatmap.pdf`

**Data sources:** Local `mathlib4/Mathlib/` for module graph, HuggingFace for declaration and namespace graphs.

Each level uses a different heatmap colormap: Blues (module), Reds (declaration), Greens (namespace). Community graphs use `tab20` for categorical node colors. Heatmaps use diagonal-maximizing reorder for visual clarity.

## Output Directory Structure

```
paper/
  analysis/          # All analytical figures (degree, DAG, robustness, centrality, containment)
  figures/           # Community detection figures (graphs + heatmaps)
```

## Regenerating All Figures

Run scripts in order of speed (fast first):

```bash
python src/plots/plot_containment.py              # instant (hardcoded data)
python src/plots/plot_namespace_degree_dist.py    # ~2 min (HF download + plot)
python src/plots/regen_figures.py                 # ~2 min (HF download + plot)
python src/plots/plot_centrality_scatter.py       # ~10 min (HF + betweenness)
python src/plots/plot_dag_depth.py                # ~5 min (HF + .lean scan + SCC)
python src/plots/plot_robustness_curves.py        # ~10 min (.lean + HF + simulation)
python src/plots/plot_community_figures.py        # ~20 min (.lean + HF + Louvain x3)
```
